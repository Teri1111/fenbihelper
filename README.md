# 刷题截图发豆包

一个最小可行的原生 Android 工具：

1. 在首页引导悬浮窗 / 通知 / 截图授权
2. 启动前台服务显示悬浮按钮
3. 单击悬浮按钮后截图
4. 将截图保存为可分享图片
5. 优先分享给豆包，失败时回退系统分享面板

## 当前实现

- `MainActivity`：权限引导、状态展示、手动测试截图、启动/停止悬浮服务
- `FloatingWindowService`：前台服务 + 可拖动悬浮按钮
- `ScreenCaptureManager`：MediaProjection 截图 + PNG 保存 + `content://` Uri
- `ShareManager`：豆包目标识别 + 系统分享回退

## 本地验证

在 Android Studio 中直接运行 `app` 模块即可。首次使用建议按以下顺序：

1. 打开悬浮窗权限设置
2. Android 13+ 申请通知权限
3. 申请截图授权
4. 点击“手动测试截图并分享”验证链路
5. 启动悬浮按钮服务

## 说明

- 截图授权当前按“当前进程缓存 + 历史状态提示”实现；如果系统回收进程或授权失效，需要重新授权。
- 截图文件保存在应用缓存目录，通过 `FileProvider` 分享。
- 长按悬浮按钮可关闭前台服务。
